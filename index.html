HTML

  • HTML (Hypertext Markup Language) elements define the content and framework.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An HTML developer is responsible for providing the structure for the webpage and making sure it provides a positive user experience.

CSS

  • CSS (Cascading Style Sheets) add style to our webpage.
  • CSS defines the colors, aesthetics, visual layout, and organization.
  • A selector defines the element or attributes to which the rules/declarations will apply.
  • Declarations contain 2 important components: the CSS property we want to apply and the value of the property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

GIT

  • GIT is a version control system that lets you manage and keep track of each version of your project
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch and adds all changes
  • git commit -m "task of what you're doing": describes what the commit contains
  • git pull origin main: receives a branches modifications into the local environment, indicates source of pull will be in the GitHub repo, Main branch
  • git push origin branch name: pushes changes locally to our remote GitHub branch

JavaScript

  • JavaScript (JS) is a programming language that allows users to interact with websites.
  • It can power your website with effects, interactivity, data communication, and much more.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • console.log is a method to output a message to the web console by adding an argument. Need to use Chrome DevTools.
